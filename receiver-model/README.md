# Receiver Model — 受け手の状態

受け手（コミュニケーションの相手）の状態を定義するレイヤー。
同じ思考・同じ表現でも、受け手が異なれば伝え方の調整が必要になる。

## 位置づけ

receiver-model は「誰に伝えるか」を決める。
thinking-mode（視点）、output-mode（チャネル）とは独立しており、自由に組み合わせ可能。

## 使い方

受け手の状態を観察し、最も近いモデルを選ぶ。不明な場合は省略し、途中で判明した時点で調整する。

## 構成

### Core（最小・頻出）

最も遭遇頻度が高く、まず検討すべきモデル。

| ファイル | 受け手の状態 | 選択シグナル |
|---|---|---|
| [busy.md](busy.md) | 忙しい・時間がない | 時間がない、結論を急いでいる |
| [aligned.md](aligned.md) | 文脈が共有されている | 前提が共有済み、本題に入れる |
| [exploratory.md](exploratory.md) | 探索的・方向未定 | 方向が定まっていない、選択肢を探している |

### 拡張

特定の状態がより強く出ている場合に使用する。

| ファイル | 受け手の状態 | 選択シグナル |
|---|---|---|
| [anxious.md](anxious.md) | 不安・リスクに敏感 | 不安が強い、リスクに敏感 |
| [delegating.md](delegating.md) | 任せたい・判断を委ねたい | 任せたい、判断を委ねたい |

## 受け手の状態として定義するもの

- 知識・前提：受け手が持っている知識レベルと共有済みの情報
- 関心・ニーズ：受け手が求めていること、重視するポイント
- 状態・コンテキスト：受け手が置かれている状況、感情的な状態
- 調整ルール：この受け手に合わせた伝え方の調整

## 拡張ルール

新しい受け手のモデルを追加する場合は、このレイヤーにファイルを追加する。
core の原則、thinking-mode の視点、output-mode の文体をここに混ぜないこと。
