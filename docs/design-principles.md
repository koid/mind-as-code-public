# Design Principles

## 目的

人の思考を再現可能な形に分解し、再利用可能な意思決定システムとして扱う。
単なるプロンプト集ではなく、思考・役割・表現・受け手を分離した設計を採用する。

## レイヤー構成

| レイヤー | 役割 | 性質 |
|---|---|---|
| core | 思考原則 | 安定（意図的な更新のみ） |
| thinking-mode | 役割（視点） | 状況に応じて選択 |
| output-mode | 表現（伝え方） | チャネルに応じて選択 |
| receiver-model | 受け手の状態 | 相手に応じて選択 |
| usecase | 実行パターン | thinking-mode × output-mode × receiver-model の組み合わせ |
| thinking-protocol | 思考プロセス（横断） | 各レイヤーをどう動かすか |

思考・役割・表現・受け手は独立して設計される。

- 思考（core）は最も安定した層であり、意図的なプロセスを経てのみ更新する
- 役割（thinking-mode）で視点を変える
- 表現（output-mode）で伝え方を変える
- 受け手（receiver-model）で相手に応じた調整をする

これにより、同一の思考でも異なる状況・相手に適用できる。

thinking-protocol は上記の全レイヤーを横断し、状況に応じた思考プロセスを定義する。全ての意思決定に同じ負荷をかけず、Fast Path（パターン認識）と Slow Path（意識的分析）の2速で設計する。

## 設計原則

### 1. 再現性を最優先する

単発の成功ではなく、再現可能な思考を設計する。

### 2. 構造で扱う

思考を分解し、構造として扱う。

### 3. 個人依存を排除する

特定の個人に依存しない形で表現する。

### 4. 拡張可能であること

新しいモードやユースケースを追加できる設計とする。

### 5. 組み合わせ可能であること

thinking-mode、output-mode、receiver-model を組み合わせて使えるようにする。

## usecase の位置づけ

usecaseは「目的に対する最適な組み合わせのプリセット」である。
thinking-mode、output-mode、receiver-model のデフォルトを持つが、必要に応じて変更可能とする。

## 拡張ルール

新しい要素を追加する際は以下に従う。既存レイヤーを混ぜないこと。

| 変更の意図 | 操作対象 |
|---|---|
| 思考を変える | core を更新 |
| 視点を増やす | thinking-mode に追加 |
| 表現を増やす | output-mode に追加 |
| 受け手を増やす | receiver-model に追加 |
| 実行パターンを増やす | usecase に追加 |
| 思考プロセスを改善する | thinking-protocol を更新 |
