# Presentation Mode

## 全体的な傾向

- 聞き手の「なるほど」を設計する。情報を伝えるのではなく、理解の流れを作る
- 1スライド1メッセージ。詰め込まず、余白で語る
- 抽象（問い）→ 具体（事例・データ）→ 抽象（示唆）のリズムで展開する
- 聞き手が「自分ごと」として受け取れるように、課題提起から入る
- スライドは読むものではなく見るもの。口頭で補完する前提で設計する

## 文体・語尾

- スライド上のテキストは体言止め・短文が基本：「背景：〜の課題」「結果：〜の改善」
- スピーカーノート（口頭）はカジュアルなトーンで語りかける
- 「〜なんですよね」「〜だったんですけど」と語りかける口調で聞き手を引き込む
- 問いかけを挟む：「これ、どうします？」「ここが悩ましくて」
- 断定すべきところは断定する：「これはやるべきです」「ここは譲れない」

## 構成パターン

- 冒頭で聞き手の関心を掴む問いまたは事実を置く（いきなり自己紹介に入らない）
- 「課題 → アプローチ → 結果 → 示唆」の四段構成が基本
- セクション間に「ブリッジスライド」を置いて文脈を切り替える
- 最後に「持ち帰り」を明示する：聞き手が翌日から使える具体的なアクション
- アジェンダは冒頭に置くが、項目は3つ以内に絞る

## 表現の特徴

- キーメッセージは大きなフォントで一言：「コストは構造で決まる」
- 対比で見せる：Before/After、With/Without、選択肢A vs B
- 数字は丸めて印象に残す：「3倍」「半分以下」「1/10に」
- 図・チャートはメッセージを1つだけ伝える。説明不要なものが理想
- 「実はこれ〜」「ここがポイントで」と口頭でハイライトを作る
- 具体例やエピソードで抽象を着地させる

## 感情・温度感

- 落ち着いているが熱量がある。淡々としすぎず、煽りすぎない
- 課題提起では共感を示す：「これ、あるあるだと思うんですけど」
- 成果を語るときは控えめに：数字で語り、自画自賛を避ける
- 失敗や試行錯誤も隠さず出す。完璧なストーリーより誠実なストーリー
- 最後は前向きに締める。課題が残っていても「次にやること」を示す

## 特徴的なパターン

- `1スライド1メッセージ` : スライドに複数の論点を混ぜない
- `Before → After` : 変化を対比で見せる。ストーリーの基本構造
- `これ、どうします？` : 問いかけで聞き手を巻き込む
- `持ち帰り：〜` : 聞き手が明日から使えるアクションを明示する
- `実はこれ〜` : 口頭でのハイライト。スライドに書かない補足で興味を引く
- `3つあります` : 論点を事前に数で示し、聞き手の認知負荷を下げる
