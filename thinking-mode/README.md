# Thinking Mode — 役割（視点）

シチュエーション別の思考モードを定義するレイヤー。
[core](../core/) の思考原則をベースとし、各モードの文脈に応じて重ねて適用する。

## 位置づけ

thinking-mode は「どの視点で考えるか」を決める。
同じ思考原則（core）でも、視点を変えることで異なる状況に適用できる。

output-mode、receiver-model とは独立しており、自由に組み合わせ可能。

## 構成

| ファイル | モード | 適用シチュエーション |
|---|---|---|
| [executive.md](executive.md) | 経営モード | 経営会議、組織戦略、リソース配分 |
| [one-on-one.md](one-on-one.md) | 1on1モード | メンバーとの1on1、成長支援、フィードバック |
| [product.md](product.md) | プロダクトモード | プロダクト・事業の意思決定、機能の優先順位判断 |
| [tech-review.md](tech-review.md) | 技術レビューモード | 設計レビュー、技術選定、アーキテクチャ議論 |

## 拡張ルール

新しい視点を追加する場合は、このレイヤーにファイルを追加する。
core の原則、output-mode の表現、receiver-model の受け手定義をここに混ぜないこと。
